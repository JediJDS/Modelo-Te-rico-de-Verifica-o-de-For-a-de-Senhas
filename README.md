# Modelo-Te-rico-de-Verifica-o-de-For-a-de-Senhas
Este projeto foi desenvolvido como parte do Projeto Integrador do curso de Tecnologia em Segurança da Informação. O objetivo é apresentar, de forma simples e prática, como avaliar a força de uma senha sem utilizar programação. É um guia básico para ajudar qualquer pessoa a entender o que torna uma senha fraca ou forte e como melhorar a segurança das suas contas.
O que é uma senha forte?
Uma senha forte é aquela que dificulta ao máximo que alguém consiga adivinhá-la, seja tentando manualmente, seja usando programas automáticos. Quanto mais variada e menos previsível for a senha, melhor para a segurança do usuário.
Em geral, uma senha forte tem:
•	tamanho adequado,
•	mistura de letras, números e símbolos,
•	e não usa informações óbvias, como nomes, datas ou palavras comuns.
Por que senhas fracas são perigosas?
Senhas simples são muito fáceis de quebrar. Isso permite que criminosos tenham acesso a e-mails, redes sociais, dados pessoais, fotos, contas bancárias e outros serviços importantes.
Muitas invasões acontecem porque:
•	a senha é curta,
•	segue padrões previsíveis (como 123456),
•	ou reutiliza informações pessoais facilmente descobertas.
Entender como avaliar e melhorar uma senha é um dos passos mais básicos da Segurança da Informação.
Como avaliar a força de uma senha (método simples)
Para facilitar o entendimento, este projeto utiliza uma pontuação teórica baseada em quatro critérios:
1.	Tamanho da senha:
Senhas curtas são mais fracas. Senhas mais longas são mais resistentes a ataques automáticos.
2.	Variedade de caracteres:
Quanto mais tipos de caracteres a senha tiver (maiúsculas, minúsculas, números e símbolos), mais difícil ela se torna.
3.	Previsibilidade:
Se a senha usa palavras comuns ou informações pessoais (nome, time de futebol, data de aniversário), ela fica fraca.
4.	Padrões repetitivos ou sequências:
Senhas como “123456”, “111111”, “abcdef” ou “qwerty” são extremamente fracas, mesmo que tenham vários caracteres.
O objetivo não é criar uma ferramenta avançada, mas oferecer uma forma fácil de qualquer pessoa olhar uma senha e identificar se ela precisa ser melhorada.
Exemplo prático de avaliação
A seguir, alguns exemplos simples usando o método:
•	Senha: joao123
É curta, usa nome próprio e termina com números previsíveis.
Resultado: senha fraca.
•	Senha: Futebol!2024
É mais longa e mistura letras, números e símbolo, apesar de usar uma palavra comum.
Resultado: senha forte.
•	Senha: 12345678
É uma sequência numérica muito conhecida.
Resultado: senha muito fraca.
Esses exemplos servem apenas para demonstrar como pensar sobre a segurança de uma senha no dia a dia.
Como criar senhas melhores
Algumas recomendações simples ajudam muito:
•	Use senhas com 12 caracteres ou mais.
•	Misture letras maiúsculas, minúsculas, números e símbolos.
•	Evite usar palavras conhecidas ou informações pessoais.
•	Nunca use a mesma senha em vários sites.
•	Prefira utilizar um gerenciador de senhas, que ajuda a guardar e criar senhas seguras.
•	Sempre habilite dupla autenticação (2FA).
Essas práticas são fáceis de adotar e aumentam bastante a segurança das contas.
Conclusão
Este projeto apresenta um modelo simples e teórico para avaliar a força de senhas. Mesmo sem utilizar programação, é possível entender como senhas fracas colocam usuários em risco e como melhorar a proteção de contas. A criação de senhas fortes é um dos fundamentos mais importantes da Segurança da Informação, sendo um hábito essencial tanto para usuários comuns quanto para profissionais da área.
